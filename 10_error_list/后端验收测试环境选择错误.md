# 2026-06-07 后端验收测试环境选择错误

## 错误分类

- 类型：测试执行错误
- 严重级别：P3
- 关联变更：`8_change_list/llm-node-eval-judge-rule-v1`
- 责任归类：验收测试流程错误，不属于业务代码缺陷

## 错误现象

在 `llm-node-eval-judge-rule-v1` 真实验收中，测试人员使用 `maven:3.9.11-eclipse-temurin-21` Docker 容器执行后端 `mvn clean test`。该容器没有访问宿主机 Docker Desktop 的能力，也没有挂载 Docker socket，导致依赖 Testcontainers 的数据库集成测试被跳过。

测试输出中出现 `10 skipped`。这些 skipped 用例覆盖 Flyway 空库迁移、PostgreSQL 表结构、事务边界等数据库集成测试能力。

## 错误原因

测试人员没有优先使用项目已配置的本机 JDK/Maven 工具链，也没有先确认 Maven Docker 容器是否具备 Docker 访问能力。

Testcontainers 需要可用 Docker 环境来临时启动测试容器。无 Docker 访问能力的 Maven 容器只能运行普通 Java 测试，不能完整执行数据库集成测试。

## 影响

- 后端自动化测试结果不能被视为完整通过。
- `0 failures / 0 errors` 不能代表数据库集成测试已执行。
- 该问题若被误判，可能错误地把测试环境限制写成开发侧业务代码缺陷，或漏掉真实数据库集成测试覆盖风险。

## 正确做法

优先使用本机环境，之后再考虑使用标准工具链执行后端测试，并确认 Docker Desktop 正常运行。标准工具链目录为：

- JDK：`D:\mytools\jdk\jdk-21.0.11+10`
- Maven：`D:\mytools\maven\apache-maven-3.9.11`

推荐命令：

```powershell
$env:JAVA_HOME = 'D:\mytools\jdk\jdk-21.0.11+10'
$env:MAVEN_HOME = 'D:\mytools\maven\apache-maven-3.9.11'
$mvnBin = "$env:MAVEN_HOME\bin"
$env:PATH = "$env:JAVA_HOME\bin;$mvnBin;$env:PATH"

cd D:\myproject\MyAgent\11_code\backend
mvn clean test
```

如果必须使用 Maven Docker 容器，必须挂载宿主机 Docker socket 或配置等价 Docker 访问能力，并先确认容器内 Testcontainers 可以访问 Docker。

## 防止复发规则

1. 后端验收前必须阅读 `12_test_env/test_env.md`。
2. 不得因为系统 `PATH` 中没有 `mvn` 就直接改用临时 Maven Docker 容器。
3. 涉及 Testcontainers 的测试必须确认测试环境可访问 Docker。
4. 验收报告必须同时记录 `Tests run`、`Failures`、`Errors`、`Skipped`。
5. 只要 Testcontainers 相关测试 skipped，后端数据库集成测试就只能标记为“未完整执行”或“测试环境限制”，不得标记为完整通过。
6. skipped 的 Testcontainers 测试必须在合规环境复跑后，才能形成最终后端自动化测试结论。
