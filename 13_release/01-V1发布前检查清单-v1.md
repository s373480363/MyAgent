# V1 发布前检查清单 v1

## 1. 代码与构建

- [x] 后端工程可执行 `mvn -q test`。
- [x] 前端工程可执行 `npm test -- --run`。
- [x] 前端工程可执行 `npm run build`。
- [x] 前端 API 类型消费来自 OpenAPI 生成类型。
- [x] 前端不存在长期手写业务 DTO 真相。

## 2. OpenAPI 与契约

- [x] `/v3/api-docs` 是唯一契约来源。
- [x] `frontend/openapi/myagent-openapi.json` 由 `/v3/api-docs` 下载生成。
- [x] `frontend/src/api/generated/schema.ts` 由 OpenAPI 快照生成。
- [x] 可执行 `npm run openapi:refresh`。
- [x] 可执行 `npm run openapi:check`，验证 OpenAPI 产物刷新后仍保持稳定。

## 3. 数据库与迁移

- [x] V1 核心表由 Flyway 迁移创建。
- [x] 工作流版本、运行、Trace、Eval 相关表已纳入迁移。
- [x] 在具备 Docker 的环境完成 PostgreSQL Testcontainers 集成测试。
- [ ] 在目标部署数据库执行 Flyway 迁移并确认版本表状态。

## 4. 后端能力

- [x] Schema 管理接口已实现。
- [x] Java 方法与工具目录接口已实现。
- [x] 外部 Agent 创建、编辑、启停、测试、secrets 接口已实现。
- [x] Settings 白名单接口已实现。
- [x] Agent 管理与 WorkflowVersion 接口已实现。
- [x] Run 与 Trace 查询接口已实现。
- [x] EvalSuite、EvalCase、EvalRun、EvalCaseResult 接口已实现。
- [ ] 在目标环境完成真实 OpenAI 调用验收。

## 5. 前端能力

- [x] Settings 页面可查询和更新白名单设置。
- [x] Schemas 页面支持列表、创建、详情、编辑 DRAFT、新版本创建。
- [x] External Agents 页面支持创建、编辑、启停、测试、secrets 单独维护。
- [x] Agents 页面支持创建、编辑、启停、详情、版本入口。
- [x] Workflow 页面支持画布、节点库、属性面板、保存、校验、发布、历史只读和复制草稿。
- [x] Debug 页面支持发起调试运行并展示绑定版本。
- [x] Runs 页面支持运行详情、NodeRun、Trace、父子运行、版本回看、NodeRun 生成用例入口。
- [x] Evals 页面支持套件、用例、运行、结果明细、历史对比。

## 6. 联调架构

- [x] 浏览器默认通过相对路径访问 `/api`。
- [x] Vite dev server 已代理 `/api`、`/v3/api-docs`、`/swagger-ui.html`、`/swagger-ui`、`/actuator`。
- [x] `VITE_API_BASE_URL` 仅作为显式覆盖项存在，不是默认真相。
- [x] 按标准启动方式在真实浏览器中验证页面不再出现 `Failed to fetch`。

## 7. 质量闸门

- [x] `runId` 使用 `agent_run.run_no`。
- [x] `evalRunId` 使用 `eval_run.run_no`。
- [x] `nodeRunId` 使用 `node_run.id`。
- [x] WorkflowVersion 的 runtimeOptions 和 referencedSchemaVersions 作为版本快照处理。
- [x] 外部 Agent 敏感 Header 不回显明文，不使用掩码保留协议。
- [x] Eval 执行顺序为 outputSchema 校验、确定性断言、可选 LLM 评分。
- [x] 完成真实前后端联调数据回溯检查。
