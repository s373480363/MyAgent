# V1 部署说明 v1

## 1. 部署组件

- 后端：`11_code/backend`
- 前端：`11_code/frontend`
- 数据库：PostgreSQL
- 迁移工具：Flyway
- 可选外部依赖：OpenAI API、Codex CLI、OpenCode CLI、自定义外部 Agent 服务

## 2. 后端前提

- JDK 21。
- Maven 3.9.x。
- PostgreSQL 可访问。
- 本地或目标环境需要配置数据库连接。
- 如果要执行真实 LLM 节点，需要配置 OpenAI-compatible API Key。禁止把 Key 写入 `system_setting` 或前端设置页。
- 默认模型网关地址为 OpenAI 官方地址；如使用 Poe，需要通过 `SPRING_AI_OPENAI_BASE_URL=https://api.poe.com` 切换，不要配置成 `https://api.poe.com/v1`。

## 3. 后端验证命令

完整后端验证需要 Docker Desktop 可用，因为 PostgreSQL Testcontainers 会启动真实 `postgres:16-alpine` 容器并执行 Flyway 迁移。

```powershell
$env:JAVA_HOME='D:\myproject\MyAgent\.tools\jdk21\jdk-21.0.11+10'
$env:Path='D:\myproject\MyAgent\.tools\jdk21\jdk-21.0.11+10\bin;D:\myproject\MyAgent\.tools\maven-3.9.11\apache-maven-3.9.11\bin;' + $env:Path
cd D:\myproject\MyAgent\11_code\backend
mvn -q test
```

## 4. 后端启动

```powershell
cd D:\myproject\MyAgent\11_code\backend
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

真实模型环境变量示例：

```powershell
$env:OPENAI_API_KEY='<OpenAI-compatible API Key>'
$env:SPRING_AI_OPENAI_BASE_URL='https://api.poe.com'
$env:MYAGENT_OPENAI_DEFAULT_MODEL='gpt-5.4-mini'
```

注意：

- `local` profile 会启用 Flyway 并依赖 PostgreSQL。
- 默认配置也会启用 Flyway；启动后端时会按迁移脚本自动检查并迁移表结构。
- 没有 PostgreSQL 时，后端不能按 local profile 完整启动。
- 数据库迁移必须由 Flyway 执行，不允许手工维护业务表。
- Poe Chat Completions 可用于真实模型调用验收，但官方文档说明 `response_format` 会被忽略；如必须验收供应商强制结构化输出，需要使用支持 `response_format` 的兼容服务或单独适配 Poe Responses API。

## 5. 前端验证命令

```powershell
cd D:\myproject\MyAgent\11_code\frontend
npm test
npm run build
```

## 6. 前端启动

```powershell
cd D:\myproject\MyAgent\11_code\frontend
npm run dev -- --host 127.0.0.1 --port 5173
```

如果后端不是 `http://localhost:8080`，需要通过 `VITE_API_BASE_URL` 指定后端地址。

## 7. 发布前联调顺序

1. 启动 PostgreSQL。
2. 启动后端并确认 Flyway 迁移成功。
3. 打开 `/v3/api-docs` 确认 OpenAPI 输出正常。
4. 启动前端。
5. 依次验证 Settings、Schemas、Agents、Workflow、Debug、Runs、Evals。
6. 从 Runs 页面用 `node_run.id` 创建 EvalCase。
7. 执行 EvalRun 并验证结果明细和历史对比。

## 8. 敏感配置要求

- OpenAI-compatible API Key 属于部署配置，不进入 `/api/settings`。
- 外部 Agent 敏感 Header 只能通过 `/api/external-agents/{adapterId}/secrets` 维护。
- 详情接口不得回显敏感值或掩码值。
