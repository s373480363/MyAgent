# Agent Studio V1 部署说明

## 1. 部署组件

- Web：`11_code/frontend`
- API：`11_code/backend`
- 数据库：PostgreSQL
- 迁移工具：Flyway

## 2. 正式部署前提

- Docker Desktop 可用
- 宿主机可访问 [http://127.0.0.1:18080](http://127.0.0.1:18080)
- 已设置 `AGENT_STUDIO_POSTGRES_PASSWORD`
- 已设置 `AGENT_STUDIO_SECRET_KEY`，且值为 Base64 编码的 32 字节随机值
- 已设置非空的 `AGENT_STUDIO_OPENAI_API_KEY`

可选模型变量：

- `AGENT_STUDIO_OPENAI_BASE_URL`，默认 `https://api.openai.com`
- `AGENT_STUDIO_OPENAI_DEFAULT_MODEL`，默认 `gpt-4.1-mini`

旧 OpenAI 变量不是正式入口；如在非测试启动路径检测到，API 会直接 fail-fast。

## 3. 标准部署命令

```powershell
cd D:\myproject\MyAgent
$env:AGENT_STUDIO_POSTGRES_PASSWORD='agent_studio_dev_password'
$env:AGENT_STUDIO_SECRET_KEY='<base64-32-byte-secret>'
$env:AGENT_STUDIO_OPENAI_API_KEY='<your-openai-api-key>'
docker compose up -d --build
```

部署拓扑：

- `web`：宿主机 `18080` -> 容器 `80`
- `api`：仅在 Compose 网络内暴露 `8080`
- `postgres`：仅在 Compose 网络内暴露 `5432`

部署变量 fail-fast 契约：

- 缺失 `AGENT_STUDIO_POSTGRES_PASSWORD` 时，`docker compose config` 必须失败。
- 缺失 `AGENT_STUDIO_SECRET_KEY` 时，`docker compose config` 必须失败。
- 缺失 `AGENT_STUDIO_OPENAI_API_KEY` 时，`docker compose config` 必须失败。
- `AGENT_STUDIO_SECRET_KEY` 不能由应用在正式 Docker 部署中自动生成，也不能使用固定默认值。

数据库契约：

- 数据库名：`agent_studio`
- 用户名：`agent_studio`
- API JDBC URL：`jdbc:postgresql://postgres:5432/agent_studio`

## 4. 部署后验收

健康检查：

```powershell
curl.exe -sS http://127.0.0.1:18080/actuator/health
```

接口联通：

```powershell
curl.exe -sS http://127.0.0.1:18080/api/settings
curl.exe -sS http://127.0.0.1:18080/v3/api-docs
```

OpenAPI 正式入口校验：

- `/v3/api-docs` 返回的 `servers[0].url` 必须是 `http://127.0.0.1:18080`
- 不允许退化成 `http://127.0.0.1`
- 不允许暴露内部地址 `http://api:8080`

数据库校验：

```powershell
docker compose exec postgres psql -U agent_studio -d agent_studio -c "select version from flyway_schema_history order by installed_rank;"
docker compose exec postgres psql -U agent_studio -d agent_studio -c "select setting_key from system_setting order by setting_key;"
```

数据库验收口径：

- fresh install 只要求 Flyway 迁移完成。
- fresh install 时 `system_setting` 允许为空。
- 若 `system_setting` 已有记录，则 `setting_key` 必须全部为 `agent.studio.*`。
- 当前运行态证据以 `/api/settings` 返回 7 个白名单项为主，不要求通过空表现场补种子数据。

## 5. OpenAPI 产物

唯一契约来源是 `/v3/api-docs`。

前端生成产物：

- `11_code/frontend/openapi/agent-studio-openapi.json`
- `11_code/frontend/src/api/generated/schema.ts`

刷新命令：

```powershell
cd D:\myproject\MyAgent\11_code\frontend
npm run openapi:refresh
```

如需从开发态 API 刷新，可显式设置：

```powershell
$env:AGENT_STUDIO_OPENAPI_BASE_URL='http://127.0.0.1:8080'
```

## 6. 开发态脚本说明

PowerShell 脚本仅用于开发态，不是正式部署入口：

- `11_code/scripts/start-backend-local.ps1`
- `11_code/scripts/start-frontend-dev.ps1`

Vite dev server 也仅用于开发联调，不构成正式发布方式。
