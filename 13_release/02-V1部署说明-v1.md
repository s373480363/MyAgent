# V1 部署说明 v1

## 1. 部署组件

- 后端：`11_code/backend`
- 前端：`11_code/frontend`
- 数据库：PostgreSQL
- 迁移工具：Flyway
- 可选外部依赖：OpenAI-compatible API、Codex CLI、OpenCode CLI、自定义外部 Agent 服务

## 2. 标准环境前提

- JDK `21`
- Maven `3.9.11`
- Node.js `24.x`
- npm `11.x`
- PostgreSQL 可访问
- Docker Desktop 可用

仓库标准做法依赖 `.tools` 下的 JDK 21 与 Maven 3.9.11，不假设系统默认 `java` 和 `mvn` 可直接用于本项目。

## 3. 后端启动

标准入口：

```powershell
cd D:\myproject\MyAgent
powershell -NoProfile -ExecutionPolicy Bypass -File .\11_code\scripts\start-backend-local.ps1
```

如需显式覆盖数据库或端口，可先设置：

```powershell
$env:MYAGENT_DATASOURCE_URL='jdbc:postgresql://127.0.0.1:5432/myagent'
$env:MYAGENT_DATASOURCE_USERNAME='myagent'
$env:MYAGENT_DATASOURCE_PASSWORD='myagent'
$env:MYAGENT_SERVER_PORT='8080'
```

真实模型环境变量示例：

```powershell
$env:OPENAI_API_KEY='<OpenAI-compatible API Key>'
$env:SPRING_AI_OPENAI_BASE_URL='https://api.poe.com'
$env:MYAGENT_OPENAI_DEFAULT_MODEL='gpt-5.4-mini'
```

说明：

- `local` profile 会启用 Flyway，并要求 PostgreSQL 可访问。
- 数据库迁移必须由 Flyway 执行，不允许手工维护业务表。
- Poe Chat Completions 可用于真实模型调用验收，但官方文档说明 `response_format` 会被忽略；如必须验收供应商强制结构化输出，需要使用支持 `response_format` 的兼容服务或单独适配 Poe Responses API。

## 4. 前端启动

标准入口：

```powershell
cd D:\myproject\MyAgent
powershell -NoProfile -ExecutionPolicy Bypass -File .\11_code\scripts\start-frontend-dev.ps1
```

标准联调语义：

- 浏览器默认只访问相对路径 `/api`
- Vite dev server 代理 `/api`、`/v3/api-docs`、`/swagger-ui.html`、`/swagger-ui`、`/actuator`
- 代理目标默认是 `http://127.0.0.1:8080`

覆盖模式：

- `MYAGENT_BACKEND_DEV_TARGET` 只用于本地 dev server 代理目标的临时覆盖，适用于标准 `8080` 端口被其他进程占用时的开发侧联调
- 该变量只改变 Vite 代理后端，不改变浏览器默认通过相对路径 `/api` 访问后端的正式口径
- `VITE_API_BASE_URL` 只用于特殊联调环境
- 它不是标准开发或部署前提，不应用来掩盖默认联调路径未闭合的问题

## 5. OpenAPI 产物

唯一契约来源是后端 `/v3/api-docs`。

前端保留两个生成产物：

- `frontend/openapi/myagent-openapi.json`
- `frontend/src/api/generated/schema.ts`

刷新命令：

```powershell
cd D:\myproject\MyAgent\11_code\frontend
npm run openapi:refresh
```

如后端不在默认 `http://127.0.0.1:8080`，可先设置：

```powershell
$env:MYAGENT_OPENAPI_BASE_URL='http://127.0.0.1:18080'
```

一致性检查：

```powershell
cd D:\myproject\MyAgent\11_code\frontend
npm run openapi:check
```

## 7. 发布前联调顺序

1. 启动 PostgreSQL，并确认数据库地址与后端配置一致。
2. 启动后端并确认 Flyway 迁移成功。
3. 打开 `/v3/api-docs`，确认 OpenAPI 输出正常。
4. 执行 `npm run openapi:check`，确认 OpenAPI 快照与生成类型同源且幂等。
5. 启动前端。
6. 依次验证 `Settings`、`Schemas`、`Agents`、`Workflow`、`Debug`、`Runs`、`Evals`。
7. 从 `Runs` 页面用 `node_run.id` 创建 EvalCase。
8. 执行 EvalRun，并验证结果明细、历史对比、Runs 详情、NodeRun 与 Trace 回溯。

## 8. 敏感配置要求

- OpenAI-compatible API Key 属于部署配置，不进入 `/api/settings`。
- 外部 Agent 敏感 Header 只能通过 `/api/external-agents/{adapterId}/secrets` 维护。
- 详情接口不得回显敏感值或掩码值。
