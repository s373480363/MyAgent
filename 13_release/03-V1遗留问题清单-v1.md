# V1 遗留问题清单 v1

## 1. 环境类遗留

### R01 Docker 不可用导致 Testcontainers 未在本机完整执行

- 类型：环境限制
- 状态：待目标环境补跑
- 影响：本机 `mvn -q test` 可以通过，但 PostgreSQL Testcontainers 用例会按既有跳过策略处理。
- 收口方式：在具备 Docker 的环境执行后端完整测试，确认数据库迁移与 PostgreSQL 方言行为。

### R02 真实 OpenAI 调用未在本机验收

- 类型：环境限制
- 状态：待目标环境验证
- 影响：当前本机可验证接口、结构化错误、本地回退和节点链路，但不能声明真实模型调用已完成验收。
- 收口方式：在配置真实 OpenAI API Key 的环境执行 LLM、REVIEW、SUMMARY、Eval 可选 LLM 评分链路。

## 2. 前端体验类遗留

### R03 前端 Vite chunk 体积超过 500 kB

- 类型：非阻塞优化
- 状态：可后续优化
- 影响：生产构建通过，但 Vite 输出大包体积提示。
- 原因：Ant Design、React Flow 和当前管理台页面集中打包。
- 收口方式：后续按路由动态导入拆包。

### R04 前端测试环境输出 jsdom getComputedStyle 提示

- 类型：测试环境提示
- 状态：非阻塞
- 影响：`npm test` 断言通过，退出码为 0。
- 原因：Ant Design 在测试环境调用 pseudo-elements 相关样式能力，jsdom 未完整实现。
- 收口方式：如需清理日志，可在测试 setup 中补更完整的样式 API mock。

## 3. 联调类遗留

### R05 真实前后端关键路径尚需在同一环境完整走通

- 类型：发布前联调项
- 状态：待执行
- 影响：开发侧已经通过本地自动化验证，但发布前仍需真实数据库、后端、前端同时运行验证。
- 必测路径：`Schema -> Agent -> Workflow 保存/发布 -> Debug -> Runs -> NodeRun 生成 EvalCase -> EvalRun -> 历史对比`。

## 4. 当前结论

以上遗留均不属于架构口径未定问题，也不要求新增接口。它们属于发布前环境验证和体验优化项。
