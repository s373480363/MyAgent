# Agent 管理平台 - 04 工作流画布与版本规则 v1

## 1. 工作流画布定位

工作流画布是 V1 的核心用户体验。用户通过画布定义 Agent 的内部执行流程，包括节点、边、条件、输入输出映射和节点配置。

画布不是只展示流程图，而是工作流定义的主要编辑入口。

## 2. 画布区域

| 区域 | 说明 |
|---|---|
| 左侧节点库 | 展示可拖拽节点类型。 |
| 中间画布 | 展示节点、边、布局和分支关系。 |
| 右侧配置面板 | 展示当前选中节点或边的配置项。 |
| 顶部操作栏 | 保存、校验、发布、调试、查看 JSON。 |

## 3. 节点库

V1 节点库包含：

- START
- END
- LLM
- CONDITION
- JAVA_METHOD
- EXTERNAL_AGENT
- TOOL
- REVIEW
- SUMMARY
- AGENT_CALL

## 4. 画布基础能力

- 拖拽创建节点。
- 删除节点。
- 复制节点。
- 连接节点边。
- 删除边。
- 配置边条件。
- 拖动画布。
- 缩放画布。
- 节点自动对齐。
- 保存草稿。
- 校验工作流。
- 发布工作流。
- 查看 JSON 视图。

## 5. 节点配置面板

节点配置面板根据节点类型展示不同字段。

通用字段：

- 节点名称
- 节点描述
- 输入 Schema
- 输出 Schema
- 输入映射
- 输出映射
- 超时时间
- 失败策略

LLM 类节点额外字段：

- 系统提示词
- 用户提示词模板
- 模型
- 温度
- 输出格式
- 可用工具列表

JAVA_METHOD 节点额外字段：

- Java 方法选择
- 入参映射
- 出参写回

EXTERNAL_AGENT 节点额外字段：

- 外部 Agent 适配器
- 调用方式
- 提示词模板
- 工作目录
- 会话策略
- 调试采集配置

AGENT_CALL 节点额外字段：

- 目标 Agent
- 输入映射
- 输出写回
- 调用超时

## 6. 工作流校验规则

- 必须存在且只能存在一个 START 节点。
- 必须至少存在一个 END 节点。
- 除 START 外，每个可执行节点必须至少有一个入边。
- 除 END 外，每个可执行节点必须至少有一个出边。
- CONDITION 节点必须有默认分支。
- CONDITION 节点的条件边必须有清晰条件。
- AGENT_CALL 节点不能直接调用当前 Agent 自己。
- 工作流必须配置最大运行步数。
- 工作流必须配置默认超时时间。
- LLM 节点必须有提示词模板或继承 Agent 默认提示词。
- JAVA_METHOD 节点必须引用已启用 Java 方法。
- EXTERNAL_AGENT 节点必须选择可用适配器，并配置提示词模板或固定调用指令。
- 引用 Schema 的节点必须保证输入映射和输出映射字段存在。
- 发布前必须校验工具、Java 方法、外部 Agent、目标 Agent、Schema、模型配置都存在。

## 7. 工作流版本规则

| 规则 | 说明 |
|---|---|
| 每个 Agent 只有一个当前草稿 | 草稿可编辑，可调试，不影响发布版本。 |
| 每次发布生成不可变版本 | 发布后生成新的 WorkflowVersion。 |
| 发布版本只读 | 已发布版本不可原地修改。 |
| 历史版本只读 | 历史版本可查看、复制，不可修改。 |
| 正式调用使用当前发布版本 | API 默认调用 `currentPublishedWorkflowVersionId`。 |
| 调试允许运行草稿 | 草稿可用于页面调试，但不能作为正式 API 调用版本。 |
| 运行记录绑定版本 | AgentRun 必须保存 `workflowVersionId`。 |

## 8. 发布流程

```text
编辑草稿
  -> 保存草稿
  -> 校验工作流
  -> 校验引用对象
  -> 校验 Schema 引用
  -> 发布
  -> 生成不可变发布版本
  -> 更新 Agent.currentPublishedWorkflowVersionId
```

## 9. 没有发布版本时的行为

- Agent 可以保存草稿。
- Agent 可以在调试页面运行草稿。
- Agent 不能通过正式 API 调用。
- AGENT_CALL 节点不能调用未发布 Agent。

## 10. Schema 引用规则

- 已发布工作流引用的 Schema 版本必须只读。
- Schema 变更通过新版本演进。
- 草稿工作流可以切换到新 Schema 版本。
- 历史运行记录必须保留当时使用的 Schema 版本。
