# Agent 管理平台 - 09 信息架构与验收标准 v1

## 1. 页面结构

| 页面 | 路径建议 | 主要功能 |
|---|---|---|
| 首页概览 | `/` | Agent 数量、最近运行、失败运行、快捷入口。 |
| Agent 列表 | `/agents` | 查看、创建、编辑、停用 Agent。 |
| Agent 详情 | `/agents/:agentId` | 查看基础信息、当前版本、运行入口和最近记录。 |
| 工作流设计器 | `/agents/:agentId/workflow` | 画布设计、保存、校验、发布工作流。 |
| 运行调试 | `/agents/:agentId/debug` | 输入测试数据，同步执行 Agent，查看结果。 |
| 运行记录 | `/runs` | 查看所有 AgentRun，按 Agent、状态、时间筛选。 |
| 运行详情 | `/runs/:runId` | 查看 NodeRun、TraceEvent、父子运行关系和最终输出。 |
| 工具管理 | `/tools` | 查看平台已注册工具。 |
| Java 方法管理 | `/methods` | 查看已注册 Java 方法、入参、出参和启用状态。 |
| 外部 Agent 管理 | `/external-agents` | 查看 Codex、OpenCode、自定义 CLI 和 HTTP 适配器。 |
| Schema 管理 | `/schemas` | 维护 JSON Schema、版本和引用关系。 |
| 节点验收 | `/evals` | 管理 EvalSuite、EvalCase、EvalRun。 |
| 系统设置 | `/settings` | 配置 OpenAI、默认模型、运行限制。 |

## 3. 数据保留对象

| 对象 | 关键内容 | 目的 |
|---|---|---|
| Agent 定义 | 名称、标识、描述、默认模型、状态。 | Agent 管理和调用。 |
| 工作流版本 | 节点、边、版本号、状态、发布时间。 | 画布编辑、发布、历史版本。 |
| 工具定义 | 名称、描述、参数 Schema、返回 Schema。 | 工具节点和 LLM 工具调用。 |
| Java 方法定义 | 方法标识、入参 Schema、出参 Schema、Java 类型。 | JAVA_METHOD 节点。 |
| 外部 Agent 定义 | 适配器标识、类型、默认命令或 URL、可用状态。 | EXTERNAL_AGENT 节点。 |
| Schema 定义 | Schema 标识、版本、JSON Schema、Java 类型。 | 结构契约。 |
| 运行记录 | Agent、工作流版本、输入、输出、状态、耗时。 | 同步调用和运行查询。 |
| 节点记录 | 节点 ID、节点类型、输入、输出、状态、耗时。 | 逐节点调试。 |
| 追踪事件 | 事件类型、时间、关联运行、事件内容。 | 时间线和审计。 |
| Agent 消息 | 父运行、子运行、调用方、被调用方、输入输出。 | 多 Agent 协作追踪。 |
| 节点验收集 | 绑定对象、验收目标、通过阈值、状态。 | 节点提示词验收。 |
| 节点验收用例 | 输入、参考答案、断言规则、确认状态。 | 用户定义验收标准。 |
| 节点验收运行 | 输出、断言结果、通过率、失败原因。 | 回归验收和质量对比。 |

## 4. 主流程验收标准

- 用户可以创建一个新的 Agent。
- 用户可以编辑 Agent 基础信息。
- 用户可以进入工作流设计器。
- 用户可以通过画布创建 START、LLM、END 节点并连接。
- 用户可以配置 LLM 节点提示词和模型参数。
- 用户可以添加 CONDITION 节点并配置默认分支。
- 用户可以添加 JAVA_METHOD 节点，选择已注册方法并配置入参和出参映射。
- 用户可以添加 EXTERNAL_AGENT 节点，选择 Codex 或 OpenCode 适配器并配置提示词模板。
- 用户可以添加 AGENT_CALL 节点调用平台内部已发布 Agent。
- 用户可以创建 Schema，并在节点中引用。
- 用户可以保存工作流草稿。
- 用户可以校验并发布工作流。
- 用户可以在调试页面同步运行 Agent。
- 用户可以通过 REST API 同步调用已发布 Agent。
- 用户可以查看 AgentRun、NodeRun 和 TraceEvent。
- 用户可以创建节点验收集，添加验收用例并执行验收。

## 5. 异常流程验收标准

- 工作流缺少 START 节点时不能发布。
- 工作流缺少 END 节点时不能发布。
- CONDITION 节点没有默认分支时不能发布。
- LLM 节点缺少必要提示词时不能发布。
- JAVA_METHOD 节点引用未启用方法时不能发布。
- EXTERNAL_AGENT 节点未选择适配器或缺少提示词模板时不能发布。
- AGENT_CALL 目标 Agent 未发布时不能发布或运行失败原因明确。
- OpenAI 调用失败时，运行状态为 FAILED，并记录模型调用失败事件。
- Java 方法调用失败时，运行状态为 FAILED，并记录方法调用错误摘要。
- 外部 Agent 调用失败时，运行状态为 FAILED，并记录适配器、退出码或 HTTP 状态、错误摘要。
- Schema 校验失败时，运行失败并标记字段路径。
- AI 生成的验收用例未被用户确认时，不能计入正式验收通过率。

## 6. 可用性验收标准

- 核心页面文案必须为中文。
- 错误提示必须说明用户需要修正什么。
- 工作流画布在常见桌面分辨率下可用。
- 节点、边和配置面板不能互相遮挡。
- 运行详情页面能快速定位失败节点。
- 同步调用接口成功和失败都必须返回 runId。
- 节点验收结果能展示失败用例和失败原因。

## 7. 风险与约束

| 风险 | V1 处理 |
|---|---|
| 工作流循环失控 | 强制最大步数和默认超时。 |
| Agent 调用链路过深 | 限制最大调用深度。 |
| 同步调用耗时较长 | 提供默认超时和明确状态。 |
| 外部 Agent 执行不可控 | 轻量接入，高级安全策略作为可选配置。 |
| Java 方法误调用 | 只允许调用显式注册方法。 |
| Schema 与 POJO 不一致 | 以 SchemaDefinition 作为跨层契约。 |
| 验收集质量不足 | 正式用例必须由用户定义或确认。 |
| 无登录权限 | V1 明确限定为本机或内网单用户部署。 |
