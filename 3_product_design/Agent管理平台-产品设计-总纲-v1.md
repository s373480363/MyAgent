# Agent 管理平台 - 产品设计总纲 v1

## 1. 产品定位

Agent 管理平台是一套用于创建、配置、执行、调试和管理多个 AI Agent 的平台型产品。用户可以在平台中创建 Agent 实例，为 Agent 定义内部工作流，并通过平台页面或后端接口同步调用执行。

本平台不是单个聊天助手，也不是只封装一个大模型调用接口。它的核心价值是把 Agent 拆成可配置的工作流、节点能力、结构化数据契约、运行记录和追踪事件，让用户可以快速生成不同类型的 Agent。

## 2. V1 核心目标

- 支持创建、编辑、发布和停用 Agent。
- 支持 React 工作流画布，用户可以拖拽节点、连接边、配置节点并发布工作流。
- 支持同步调用 Agent，并返回最终执行结果。
- 支持 LLM、Java 方法、外部 Agent、工具、条件、审核、总结、内部 Agent 调用等节点能力。
- 支持通过 JSON Schema 统一表达节点输入、节点输出、Java POJO、前端表单和模型结构化输出。
- 支持节点级提示词验收，正式验收集由用户定义或确认。
- 支持运行记录、节点运行记录和 Trace 事件，便于调试和审计。
- 支持本机或内网单用户部署，不做登录、权限、多租户。

## 3. 已确认关键决策

| 决策项 | V1 决策 |
|---|---|
| 前端技术 | React |
| 后端技术 | Spring Boot + Spring AI + LangGraph4j |
| 数据库 | PostgreSQL |
| 模型供应商 | OpenAI 官方 API |
| Azure OpenAI | 不进入 V1 |
| 调用方式 | 同步返回 |
| 工作流画布 | V1 必须实现 |
| 部署边界 | 本机或内网单用户 |
| 用户权限 | 不做登录、角色、权限、多租户 |
| 工作流执行 | LangGraph4j |
| AI 能力底座 | Spring AI |
| 结构化数据 | JSON Schema |
| Java 方法调用 | JAVA_METHOD 节点 |
| 外部 Agent 调用 | EXTERNAL_AGENT 节点 |
| 内部 Agent 调用 | AGENT_CALL 节点 |
| 节点验收 | 用户主导，AI 只做辅助 |

## 4. V1 开发范围

V1 开发范围包含以下能力，均属于 V1 要完成的产品目标：

- Agent 管理
- 工作流画布
- 工作流草稿、发布版本、历史版本
- START、END、LLM、CONDITION、JAVA_METHOD、EXTERNAL_AGENT、TOOL、REVIEW、SUMMARY、AGENT_CALL 节点
- OpenAI 官方 API 接入
- Java 方法注册与调用
- 外部 Agent 轻量适配
- Schema 管理
- 节点级提示词验收
- 同步运行
- Agent 间同步调用
- 运行记录
- NodeRun 记录
- TraceEvent 追踪
- REST API 调用

## 5. V1 不包含内容

- 用户登录、角色和权限控制。
- 多租户隔离。
- 公网 SaaS 安全边界。
- Azure OpenAI。
- 复杂 RAG 知识库。
- 长期记忆系统。
- 异步任务队列。
- 人工审批节点。
- 完整工作流验收。
- 插件市场。
- 自动生成权威 benchmark。
- 自动优化提示词。
- 外部 Agent 深度托管。

## 6. 默认运行限制

| 限制项 | 建议默认值 | 说明 |
|---|---:|---|
| 单次 Agent 运行超时 | 10 分钟 | Agent 级默认超时，可配置覆盖。 |
| 单个 LLM 节点超时 | 2 分钟 | LLM 节点默认超时，可配置覆盖。 |
| 单个 JAVA_METHOD 节点超时 | 30 秒 | Java 方法节点默认超时，可配置覆盖。 |
| 单个 EXTERNAL_AGENT 节点超时 | 10 分钟 | 外部 Agent 节点默认超时，可配置覆盖。 |
| 最大工作流步数 | 30 | 防止循环或条件分支导致无限执行。 |
| 最大 Agent 调用深度 | 3 | 防止 Agent 互调链路过深。 |

- 默认 OpenAI 模型的具体名称需在实现阶段根据当时可用模型确认。

- Java 方法注册方式采用注解、Spring Bean 注册还是配置注册，需要在架构设计阶段确定。
- JSONPath 映射语法需要在接口设计阶段确定。
