# 01-技术选型与架构约束 v1

## 1. 技术选型原则

- 简洁高效，避免冗余设计和过度封装。
- SchemaDefinition 是唯一跨层结构契约，禁止前端、后端、工作流和 POJO 各自维护结构。
- v1 是单体应用架构，不拆微服务。
- Trace 是核心能力，不能后补。
- 本文档只描述实现顺序建议，不改变完整 V1 架构范围。

## 2. 前端选型

| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | React + TypeScript | 支撑复杂管理台、工作流画布和动态表单 |
| 构建工具 | Vite | 本地启动快，配置简单 |
| 路由 | React Router | 管理多页面路由 |
| 服务端状态 | TanStack Query | 管理列表、详情、运行记录和发布状态 |
| 本地状态 | Zustand | 管理画布选中态、未保存状态、局部 UI 状态 |
| 画布 | React Flow | 已确认选型，负责节点、边、缩放、拖拽和自定义节点 |
| 组件库 | Ant Design | 适合中文管理台，表格、表单、弹窗、抽屉能力完整 |
| JSON 编辑 | Monaco Editor | 用于 Schema、工作流 JSON 视图、运行输入输出查看 |
| 动态表单 | JSON Schema 表单渲染组件 | START 输入、节点配置、验收用例输入共用 Schema |

## 3. 后端选型

| 类别 | 选型 | 说明 |
|------|------|------|
| 语言 | Java 21 | 配合 Spring Boot 3.x 和现代 Java 能力 |
| 框架 | Spring Boot 3.x | 单体后端、配置、事务和扩展点组织 |
| AI 接入 | Spring AI | OpenAI 官方 API 调用，后续模型扩展保持上层接口稳定 |
| 工作流执行 | LangGraph4j | 将 WorkflowVersion 编译为图执行 |
| 数据库 | PostgreSQL | 关系字段 + JSONB 混合存储 |
| 迁移 | Flyway | 管理数据库结构演进 |
| 数据访问 | MyBatis | SQL 显式，JSONB 查询可控 |
| JSON 处理 | Jackson | JSON 与 POJO 转换 |
| Schema 校验 | JSON Schema 校验库 | 统一节点输入输出校验 |
| 路径映射 | JSONPath 库 + 平台限制 | 读取使用 JSONPath 子集，写回使用受控字段路径 |
| 测试 | JUnit 5 + AssertJ + Testcontainers | 单元、集成和 PostgreSQL 测试 |

## 4. 明确不选

| 不选内容 | 原因 |
|----------|------|
| 仅 JSON 编辑器替代画布 | 产品确认 v1 必须实现工作流画布 |
| 任意反射调用 Java 方法 | 安全和稳定性风险高 |
| 脚本式条件表达式 | 带来注入和沙箱问题 |
| 内置示例 Agent、示例 Java 方法、示例 Schema | V1 不强制内置；如需降低首次使用门槛，可由用户手册或后续模板能力提供 |
| 微服务架构 | v1 本机或内网单用户部署，单体更简单可靠 |
| 公网 SaaS 安全架构 | v1 不做登录、权限、多租户 |

## 5. 架构约束

- 所有用户可见文案、错误消息、日志说明使用中文。
- 后端类、方法、字段在实现时必须按项目规范编写中文 Javadoc。
- OpenAI Key 只存在后端配置或环境变量，不通过接口返回前端。
- 模型提示词和模型输出按已确认要求默认保存全文，成立前提是本机或内网单用户部署。
- 发布版本和被引用 Schema 版本不可变。
- 调试运行可以使用草稿；正式 API 调用只能使用当前发布版本。
