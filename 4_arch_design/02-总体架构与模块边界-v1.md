# 02-总体架构与模块边界 v1

## 1. 总体架构

```text
React 管理端
  -> REST API
  -> Spring Boot 单体后端
      -> Web/API 层
      -> Application 层
      -> Domain 层
      -> Runtime 层
      -> Registry 层
      -> Infrastructure 层
  -> PostgreSQL
  -> OpenAI 官方 API
  -> Java 方法 / 工具 / 外部 Agent
```

## 2. 分层职责

| 层级 | 职责 | 禁止事项 |
|------|------|----------|
| Web/API 层 | 接收请求、参数校验、响应包装、调用应用服务 | 不写复杂业务规则，不直接执行节点 |
| Application 层 | 编排创建、发布、运行、调试、验收等用例 | 不嵌入 OpenAI、数据库、JSONPath 实现细节 |
| Domain 层 | 领域对象、状态机、发布规则、运行状态规则 | 不依赖 Web DTO 和数据库 mapper |
| Runtime 层 | 校验工作流、编译图、执行节点、写 Trace | 不修改草稿，不绕过发布版本 |
| Registry 层 | 模型、Java 方法、工具、外部 Agent 适配器目录 | 不允许页面输入任意类名、方法名、命令绕过注册 |
| Infrastructure 层 | PostgreSQL、OpenAI、Schema 校验、JSONPath、CLI/HTTP | 不承载业务流程决策 |

## 3. 模块依赖方向

```text
web -> application -> domain
application -> runtime -> domain
runtime -> registry -> domain
runtime -> infrastructure
application -> repository interface -> infrastructure mapper
domain 不反向依赖 web、runtime、infrastructure
```

## 4. 业务模块

| 模块 | 职责 |
|------|------|
| agent | AgentDefinition、Agent 创建编辑、停用、当前发布版本指针 |
| workflow | WorkflowVersion、NodeDefinition、EdgeDefinition、草稿和发布 |
| schema | SchemaDefinition、版本规则、Schema 校验 |
| runtime | WorkflowCompiler、NodeExecutor、WorkflowContext、TraceWriter |
| run | AgentRun、NodeRun、TraceEvent、运行查询 |
| method | JavaMethodDefinition、注解扫描、Java 方法执行 |
| model | OpenAI 调用封装、模型参数、结构化输出处理 |
| collaboration | AGENT_CALL、AgentMessage、父子 AgentRun |
| externalagent | EXTERNAL_AGENT、CLI/HTTP 适配器 |
| tool | ToolDefinition、ToolRegistry、ToolNodeExecutor |
| eval | EvalSuite、EvalCase、EvalRun、节点验收执行 |
| settings | 默认模型、默认超时等系统配置 |

## 5. 事务边界

- 创建、编辑、保存草稿、发布版本使用短事务。
- 同步运行不持有长事务。
- AgentRun 创建、NodeRun 写入、TraceEvent 写入、最终状态更新分别使用短事务。
- OpenAI 调用、外部 Agent 调用、Java 方法执行不能包在数据库长事务内。
- 运行失败时必须尽力写入失败 NodeRun、NODE_ERROR 和 RUN_FINISHED。

## 6. 分期对架构的影响

实现顺序只影响代码启用先后，不影响模块边界。第一批可以只启用部分 NodeExecutor，但完整 V1 的包结构、枚举、数据模型和接口分组应按完整范围设计，避免后续推翻。
