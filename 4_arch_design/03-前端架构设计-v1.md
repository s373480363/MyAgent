# 03-前端架构设计 v1

## 1. 前端定位

前端是 Agent 管理台、工作流设计器、调试台和验收中心。核心复杂度集中在 React Flow 画布、节点配置面板、Schema 动态表单和运行详情 Trace 查看。

## 2. 目录结构

```text
src/
  app/                 应用入口、路由、全局布局、QueryClient
  api/                 REST API client、响应封装、错误处理
  features/
    agents/            Agent 列表、创建、编辑、详情
    workflow/          React Flow 画布、节点库、配置面板、发布校验
    runs/              调试运行、运行列表、运行详情、Trace 时间线
    schemas/           Schema 管理、Schema 编辑、引用查看
    methods/           Java 方法目录、方法详情
    tools/             工具目录、工具详情
    externalAgents/    外部 Agent 适配器目录、适配器详情
    evals/             节点验收集、用例、验收运行
    settings/          OpenAI 默认模型、超时等系统设置
  shared/
    components/        通用布局、表格、状态标签、JSON 查看器
    forms/             JSON Schema 动态表单
    stores/            Zustand 本地状态
    types/             由 OpenAPI 生成的前端接口类型
```

## 3. 主要路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | 首页概览 | Agent 数量、最近运行、失败运行、快捷入口 |
| `/agents` | Agent 列表 | 查看、创建、编辑、停用、进入设计器 |
| `/agents/:agentId` | Agent 详情 | 基础信息、当前发布版本、最近运行 |
| `/agents/:agentId/workflow` | 工作流设计器 | React Flow 画布、保存草稿、发布校验 |
| `/agents/:agentId/debug` | 运行调试 | 草稿或发布版本调试运行 |
| `/runs` | 运行记录 | 按 Agent、状态、时间筛选 |
| `/runs/:runId` | 运行详情 | NodeRun、TraceEvent、输入输出全文 |
| `/schemas` | Schema 管理 | Schema 创建、编辑、版本查看 |
| `/methods` | Java 方法管理 | 查看已注册方法，不提供任意方法录入 |
| `/tools` | 工具管理 | 查看已注册工具 |
| `/external-agents` | 外部 Agent 管理 | 查看外部 Agent 适配器 |
| `/evals` | 节点验收 | 管理 EvalSuite、EvalCase、EvalRun |
| `/settings` | 系统设置 | 默认模型、默认超时等配置 |

## 4. React Flow 状态模型

| 状态 | 来源 | 说明 |
|------|------|------|
| React Flow nodes | WorkflowVersion.nodesJson | 节点位置、显示名称、类型、UI 元数据 |
| React Flow edges | WorkflowVersion.edgesJson | 连线、条件边、默认分支、展示元数据 |
| NodeDefinition | 节点配置面板 | 节点业务配置、输入映射、输出映射、Schema 引用 |
| Draft state | Zustand | 当前选中节点、未保存状态、校验结果 |
| Server state | TanStack Query | Agent、Schema、Java 方法、运行记录、验收结果 |

## 5. 工作流设计器布局

- 左侧节点库：START、LLM、CONDITION、JAVA_METHOD、AGENT_CALL、EXTERNAL_AGENT、TOOL、REVIEW、SUMMARY、END。
- 中间画布：React Flow 节点、边、缩放、拖动、选中态。
- 右侧配置面板：根据节点类型展示配置项。
- 顶部操作区：保存草稿、发布校验、发布、运行调试、JSON 视图。

## 6. 配置面板原则

- 配置项按基础配置、运行配置、高级配置分组。
- Schema 字段使用动态表单渲染。
- 引用对象只能从后端目录选择，例如 Java 方法、工具、目标 Agent、外部 Agent 适配器。
- 发布校验错误必须能定位到问题节点或问题边。
- 不允许在页面输入任意 Java 类名、方法名或脚本代码。

## 7. 运行详情页面

- 左侧展示本次运行经过的节点列表。
- 中间展示选中 NodeRun 的输入、输出、状态、耗时、错误。
- 右侧展示 TraceEvent 时间线。
- AGENT_CALL 节点支持跳转子运行。
- 模型提示词和模型输出按全文展示，但需要适合长文本阅读。

## 8. 前端错误处理

- 所有错误提示使用中文。
- 接口失败优先展示后端 `error.message`。
- 工作流校验失败展示节点级错误列表，并在画布上标记问题节点。
- 运行失败提供跳转运行详情入口。

## 9. 前后端类型契约

- 前后端类型不手工维护双份。
- 后端以 OpenAPI 输出接口契约。
- 前端通过 OpenAPI 生成 TypeScript API 类型和请求响应类型。
- `shared/types/` 只存放生成结果和前端局部扩展类型。
- 如果接口字段变更，应先更新后端 OpenAPI，再重新生成前端类型。
