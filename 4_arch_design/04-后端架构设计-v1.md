# 04-后端架构设计 v1

## 1. 后端定位

后端采用 Spring Boot 单体架构，负责领域模型、工作流发布、运行编排、模型供应商调用、Java 方法执行、工具调用、外部 Agent 适配、节点验收和 Trace 持久化。

## 2. 包结构

```text
com.myagent/
  common/              通用响应、错误码、分页、时间、JSON 工具
  config/              数据库、运行限制、JSON Schema 和启动迁移配置
  agent/               AgentDefinition、Agent 应用服务和接口
  workflow/            WorkflowVersion、节点定义、发布校验
  schema/              SchemaDefinition、版本规则、Schema 校验
  runtime/             WorkflowCompiler、NodeExecutor、WorkflowContext、TraceWriter
  run/                 AgentRun、NodeRun、TraceEvent、运行查询
  method/              JavaMethodDefinition、AgentMethod 注解、注册和执行
  model/               模型供应商目录、模型供应项路由、模型参数、结构化输出处理
  collaboration/       AGENT_CALL、父子 AgentRun、AgentMessage
  externalagent/       EXTERNAL_AGENT、CLI/HTTP 适配器、外部调用记录
  tool/                ToolDefinition、ToolNodeExecutor、工具调用记录
  eval/                EvalSuite、EvalCase、EvalRun、节点验收执行
  settings/            系统配置读取和页面可见配置
```

## 3. 应用服务

| 服务 | 职责 |
|------|------|
| AgentApplicationService | 创建、编辑、停用 Agent，维护当前发布版本指针 |
| WorkflowApplicationService | 保存草稿、发布校验、生成不可变发布版本 |
| RunApplicationService | 创建 AgentRun，同步执行工作流，更新最终状态 |
| RunQueryService | 查询 AgentRun、NodeRun、TraceEvent |
| SchemaApplicationService | 维护 SchemaDefinition，执行版本锁定和引用检查 |
| JavaMethodApplicationService | 展示方法目录，启用或停用方法，提供执行入口 |
| AgentCallApplicationService | 处理平台内部 Agent 同步调用、调用深度和父子运行 |
| ExternalAgentApplicationService | 管理外部 Agent 定义，调度 CLI/HTTP 适配器 |
| ToolApplicationService | 管理工具目录，执行工具节点和工具引用校验 |
| EvalApplicationService | 管理节点验收集、验收用例和验收运行 |

## 4. 注册目录

| Registry | 说明 |
|----------|------|
| ModelOfferingRegistry | 管理模型供应商、模型供应项和运行路由 |
| JavaMethodRegistry | 扫描和维护可调用 Java 方法 |
| ToolRegistry | 维护平台工具定义和工具执行器 |
| ExternalAgentRegistry | 维护外部 Agent 适配器 |
| NodeExecutorRegistry | 按节点类型路由到 NodeExecutor |

## 5. Java 方法注册

- v1 使用注解扫描 + Spring Bean 调用方式。
- 方法必须属于 Spring Bean。
- 方法必须显式标记平台注解。
- 方法入参和出参必须可序列化为 JSON。
- 方法入参推荐为一个 Java POJO，返回值推荐为一个 Java POJO 或可序列化对象。
- 方法必须关联 inputSchema 和 outputSchema。
- 前端只能选择已注册方法，不能输入任意类名和方法名。

## 6. 模型调用封装

- 统一由 model 模块通过模型供应项调用 OpenAI-compatible 供应商。
- Agent 可配置可空默认模型供应项；LLM 类节点最终调用模型时必须解析到 `modelOfferingKey`。
- LLM 类节点可覆盖模型供应项、温度和超时。
- 配置 outputSchema 时执行结构化输出校验。
- 未配置 outputSchema 时按普通文本输出处理。
- MODEL_REQUEST 和 MODEL_RESPONSE TraceEvent 默认保存全文，预留配置项控制开关。

## 7. 错误处理

- 后端错误响应统一包含 `code`、`message`、`details`。
- 用户可见消息必须为中文。
- 原始异常摘要写入 Trace，不直接把无意义堆栈暴露到普通页面。
- 同步运行失败也必须返回 runId。

## 8. OpenAPI 契约输出

- 后端负责输出 OpenAPI 文档，作为前后端接口类型的唯一来源。
- Controller 请求体、响应体和错误响应都必须纳入 OpenAPI。
- 前端 TypeScript 类型通过 OpenAPI 生成，不手工复制 Java DTO。
- 公共接口详细字段在 `7_interface_design` 中展开，但必须遵守此契约生成机制。

## 9. 事务策略

- 模型供应商、Java 方法、工具、外部 Agent 调用不包在长事务内。
- NodeRun、TraceEvent 使用短事务及时落库。
- 运行结束必须写 RUN_FINISHED。
- 运行失败必须尽力写失败节点和 NODE_ERROR。
