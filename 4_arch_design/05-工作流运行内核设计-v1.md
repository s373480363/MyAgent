# 05-工作流运行内核设计 v1

## 1. 运行内核目标

运行内核负责将发布态 WorkflowVersion 编译为 LangGraph4j 图，并在执行过程中完成节点调度、上下文流转、Schema 校验、超时控制和 Trace 写入。

## 2. 核心组件

| 组件 | 职责 |
|------|------|
| WorkflowValidator | 发布前校验节点、边、Schema、Java 方法、工具、目标 Agent、外部 Agent 引用 |
| WorkflowCompiler | 将 WorkflowVersion 编译为 LangGraph4j 图 |
| WorkflowContext | 运行时 JSON 上下文，保存输入、节点输出、中间状态和最终输出 |
| NodeExecutorRegistry | 按节点类型找到 NodeExecutor |
| NodeExecutor | 节点执行接口 |
| MappingService | 执行输入映射和输出写回 |
| SchemaValidationService | 执行节点输入输出 Schema 校验 |
| TraceWriter | 写入 NodeRun 和 TraceEvent |
| RuntimeLimitGuard | 控制超时、最大步数、最大 Agent 调用深度 |

## 3. 执行流程

```text
接收同步运行请求
  -> 确定运行版本：正式 API 使用发布版本，调试页面可使用草稿
  -> 创建 AgentRun，状态 PENDING
  -> 初始化 WorkflowContext，写入 $.input
  -> WorkflowCompiler 编译 LangGraph4j 图
  -> 更新 AgentRun 为 RUNNING
  -> 执行节点
  -> 每个节点执行前后写 NodeRun 和 TraceEvent
  -> 到达 END 节点，生成最终输出
  -> 更新 AgentRun 为 SUCCESS / FAILED / TIMEOUT
  -> 返回 runId、status、output、durationMs
```

## 4. NodeExecutor 接口职责

每个 NodeExecutor 需要完成：

- 根据 inputMapping 从 WorkflowContext 提取输入。
- 执行节点专属逻辑。
- 执行必要的 Schema 校验。
- 根据 outputMapping 写回 WorkflowContext。
- 写入 NodeRun 状态和 TraceEvent。
- 将异常转换为中文错误消息。

## 5. v1 节点执行器

| 节点 | 执行器 | 关键规则 |
|------|--------|----------|
| START | StartNodeExecutor | 校验调用输入，写入初始上下文 |
| LLM | LlmNodeExecutor | 调用 OpenAI，按 outputSchema 可选校验 |
| CONDITION | ConditionNodeExecutor | 声明式条件规则，必须有默认分支 |
| JAVA_METHOD | JavaMethodNodeExecutor | 调用已注册 Java 方法，入参出参均校验 |
| AGENT_CALL | AgentCallNodeExecutor | 创建子 AgentRun，同步调用目标 Agent |
| EXTERNAL_AGENT | ExternalAgentNodeExecutor | 调用 CLI/HTTP 外部 Agent，写回结果摘要 |
| TOOL | ToolNodeExecutor | 直接调用已注册工具 |
| REVIEW | ReviewNodeExecutor | LLM 类审核节点，输出结论、原因、评分 |
| SUMMARY | SummaryNodeExecutor | LLM 类总结节点，输出文本或结构化摘要 |
| END | EndNodeExecutor | 生成 AgentRun 最终输出 |

## 6. 默认运行限制

| 限制项 | 默认值 | 说明 |
|--------|--------|------|
| 单次 Agent 运行超时 | 10 分钟 | Agent 可覆盖 |
| 单个 LLM 节点超时 | 2 分钟 | 节点可覆盖 |
| 单个 JAVA_METHOD 节点超时 | 30 秒 | 节点可覆盖 |
| 单个 EXTERNAL_AGENT 节点超时 | 10 分钟 | 节点可覆盖 |
| 最大工作流步数 | 30 | 防止循环失控 |
| 最大 Agent 调用深度 | 3 | AGENT_CALL 强制校验 |

## 7. 运行状态

- PENDING：运行记录已创建。
- RUNNING：工作流执行中。
- SUCCESS：到达 END 节点并生成最终输出。
- FAILED：节点执行、模型调用、方法调用、工具调用或 Schema 校验失败。
- TIMEOUT：超过运行或节点超时。
- CANCELED：被系统取消或父运行超时后取消子运行。
