# 06-节点体系架构设计 v1

## 1. 节点体系原则

- 节点是工作流中最小可配置执行单元。
- 每个节点拥有独立 NodeRun。
- 节点输入输出通过 Schema 和映射表达。
- 节点能力通过 NodeExecutor 实现。
- v1 所有节点都进入统一 NodeDefinition 和 NodeExecutorRegistry。

## 2. NodeDefinition

```json
{
  "nodeId": "string",
  "type": "START | LLM | CONDITION | JAVA_METHOD | AGENT_CALL | EXTERNAL_AGENT | TOOL | REVIEW | SUMMARY | END",
  "name": "节点显示名称",
  "description": "节点职责说明",
  "inputSchemaRef": {
    "schemaKey": "string",
    "version": 1
  },
  "outputSchemaRef": {
    "schemaKey": "string",
    "version": 1
  },
  "inputMapping": {},
  "outputMapping": {},
  "timeoutSeconds": 120,
  "failurePolicy": "FAIL_FAST | USE_DEFAULT_BRANCH | CONTINUE_WITH_EMPTY_OUTPUT",
  "config": {},
  "ui": {
    "position": { "x": 0, "y": 0 }
  }
}
```

## 3. EdgeDefinition

```json
{
  "edgeId": "string",
  "sourceNodeId": "string",
  "targetNodeId": "string",
  "type": "NORMAL | CONDITION | DEFAULT | END",
  "condition": {},
  "isDefault": false,
  "description": "边说明"
}
```

## 4. 节点职责

| 节点 | 职责 | 关键配置 |
|------|------|----------|
| START | 工作流入口 | 输入 Schema、示例输入 |
| LLM | 调用 OpenAI | 系统提示词、用户提示词模板、模型、温度、输入 Schema、输出 Schema、允许工具 |
| CONDITION | 根据上下文选择分支 | 条件规则、默认分支、失败策略、说明 |
| JAVA_METHOD | 调用平台注册 Java 方法 | 方法标识、入参映射、出参映射、超时 |
| AGENT_CALL | 调用平台内部已发布 Agent | 目标 Agent、输入映射、超时、失败策略 |
| EXTERNAL_AGENT | 调用平台外部 Agent | 适配器、提示词模板、工作目录、输出写回、超时 |
| TOOL | 调用平台注册工具 | 工具标识、参数映射、失败策略 |
| REVIEW | 审核输入或上游输出 | 审核标准、通过条件、失败提示、输出 Schema |
| SUMMARY | 总结上下文或上游结果 | 总结目标、长度限制、输出格式 |
| END | 工作流出口 | 最终输出映射 |

## 5. 发布校验规则

- 必须存在且只能存在一个 START 节点。
- 必须至少存在一个 END 节点。
- 除 START 外，每个可执行节点必须至少有一个入边。
- 除 END 外，每个可执行节点必须至少有一个出边。
- CONDITION 节点必须有默认分支。
- LLM、REVIEW、SUMMARY 必须有提示词模板或可继承的 Agent 默认提示词。
- JAVA_METHOD 必须引用已注册且启用的方法。
- AGENT_CALL 必须引用已发布目标 Agent，不能直接调用当前 Agent。
- EXTERNAL_AGENT 必须引用已注册且可用的外部 Agent 适配器。
- TOOL 必须引用已注册且启用的工具。
- 所有 Schema 引用必须存在。
- 发布后引用的 Schema 版本锁定。
- 引用 Schema 的节点必须校验输入映射和输出映射字段存在，不能把不存在或不符合 Schema 的字段写入上下文。
- 节点级 timeoutSeconds 必须大于 0；未配置时使用节点类型默认超时。
- failurePolicy 必须属于节点类型支持的策略集合。

## 6. 条件规则

v1 CONDITION 节点使用声明式规则，不执行脚本。建议支持：

- 字段等于。
- 字段不等于。
- 包含 / 不包含。
- 数值比较。
- 枚举匹配。
- 默认分支。

复杂判断应通过 JAVA_METHOD 节点完成，再由 CONDITION 基于结构化结果分支。

CONDITION 的失败策略：

- `USE_DEFAULT_BRANCH`：条件无法求值时走默认分支。
- `FAIL_FAST`：条件无法求值时节点失败。

v1 默认使用 `USE_DEFAULT_BRANCH`，但节点可以显式配置为 `FAIL_FAST`。
