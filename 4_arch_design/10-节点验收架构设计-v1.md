# 10-节点验收架构设计 v1

## 1. 定位

节点验收用于验证单个 LLM 类节点的提示词、模型参数和结构化输出。v1 不做完整工作流验收。

支持节点：

- LLM。
- REVIEW。
- SUMMARY。

## 2. 领域对象

| 对象 | 职责 |
|------|------|
| EvalSuite | 绑定 Agent、WorkflowVersion、节点，保存验收目标和通过阈值 |
| EvalCase | 保存输入、参考答案、断言规则、评分规则、关键用例标记和确认状态 |
| EvalRun | 保存一次验收运行汇总结果 |
| EvalCaseResult | 保存单条用例执行结果 |
| EvalExecutor | 执行验收用例，产出结果 |

## 3. EvalSuite

关键字段：

- suiteId。
- agentId。
- workflowVersionId。
- nodeId。
- name。
- goal。
- passThreshold。
- status：DRAFT、CONFIRMED、ARCHIVED。

## 4. EvalCase

关键字段：

- caseId。
- suiteId。
- inputJson。
- referenceAnswerJson。
- assertionsJson。
- scoreRuleJson。
- critical。
- confirmStatus：USER_CREATED、USER_CONFIRMED、AI_DRAFT_PENDING、ARCHIVED。

AI 生成用例默认是 AI_DRAFT_PENDING，不能计入正式通过率。

## 5. 断言类型

- Schema 校验。
- 字段等于。
- 包含 / 不包含。
- 正则匹配。
- 数值范围。
- 枚举校验。
- JSONPath 断言。
- 可选 LLM 评分。

## 6. 执行流程

```text
选择 EvalSuite
  -> 读取绑定 WorkflowVersion 和节点定义
  -> 过滤已确认 EvalCase
  -> 逐条构造节点输入
  -> 执行目标 LLM 类节点
  -> 校验 outputSchema
  -> 执行确定性断言
  -> 可选执行 LLM 评分
  -> 创建或关联 AgentRun(runType=EVAL)，用于统一 Trace 时间线
  -> 写 EvalRun 和 EvalCaseResult
  -> 可选写 EVAL_CASE_RESULT TraceEvent
```

## 7. 通过规则

- 普通用例按通过率计算。
- 关键用例失败时，即使整体通过率达标，也应提示验收失败。
- 未确认用例不计入正式通过率。
- LLM 评分是可选辅助，不替代用户定义的正式断言。

## 8. 数据关联建议

节点验收必须以 EvalRun 和 EvalCaseResult 作为正式结果来源。

为了统一 Trace 展示，验收运行可以创建 `AgentRun(runType=EVAL)`。该 AgentRun 只承载运行上下文和 Trace 时间线，不替代 EvalRun。TraceEvent 通过 evalRunId 关联 EvalRun，避免新增未定义的 EvalTraceEvent 模型。
