# 12-公共接口架构设计 v1

## 1. 接口原则

- REST API 统一前缀为 `/api`。
- v1 不做登录鉴权。
- 所有错误消息使用中文。
- 成功和失败响应结构稳定。
- 同步运行失败也必须返回 runId。
- 详细字段在 `7_interface_design` 中定义。
- 后端必须输出 OpenAPI 契约，前端 TypeScript 类型由 OpenAPI 生成。

## 2. 响应规范

成功响应：

```json
{
  "success": true,
  "data": {},
  "error": null
}
```

失败响应：

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "WORKFLOW_VALIDATION_FAILED",
    "message": "工作流校验失败，请修正标记节点后重新发布。",
    "details": []
  }
}
```

## 3. API 分组

| 分组 | 接口方向 | 说明 |
|------|----------|------|
| Agent | `GET /api/agents`、`POST /api/agents`、`PUT /api/agents/{agentId}`、`PUT /api/agents/{agentId}/status` | Agent 管理和启用停用 |
| 工作流草稿 | `GET /api/agents/{agentId}/workflow-draft`、`PUT /api/agents/{agentId}/workflow-draft` | 每个 Agent 一个当前草稿 |
| 工作流发布 | `POST /api/agents/{agentId}/workflow-draft/validate`、`POST /api/agents/{agentId}/workflow-draft/publish` | 校验并发布不可变版本 |
| 同步运行 | `POST /api/agents/{agentKey}/runs` | 正式 API 调用当前发布版本 |
| 调试运行 | `POST /api/agents/{agentId}/debug-runs` | 可运行草稿或发布版本 |
| 运行查询 | `GET /api/runs`、`GET /api/runs/{runId}` | 查询 AgentRun、NodeRun、TraceEvent |
| Schema | `GET /api/schemas`、`POST /api/schemas`、`GET /api/schemas/{schemaId}` | Schema 管理 |
| Java 方法 | `GET /api/java-methods`、`GET /api/java-methods/{methodId}` | 查看已注册方法 |
| 工具 | `GET /api/tools`、`GET /api/tools/{toolId}` | 查看已注册工具 |
| 外部 Agent | `GET /api/external-agents`、`GET /api/external-agents/{adapterId}` | 查看外部 Agent 适配器 |
| 节点验收 | `GET /api/eval-suites`、`POST /api/eval-suites`、`POST /api/eval-suites/{suiteId}/runs` | 管理验收和执行验收 |
| 系统设置 | `GET /api/settings`、`PUT /api/settings` | 默认模型、默认超时等非密钥配置 |

## 4. 同步运行响应

```json
{
  "runId": "string",
  "agentKey": "string",
  "workflowVersionId": "string",
  "status": "SUCCESS | FAILED | TIMEOUT",
  "output": {},
  "error": {
    "code": "string",
    "message": "中文错误说明"
  },
  "durationMs": 0
}
```

## 5. 错误原则

- 工作流发布失败返回节点级错误。
- Schema 校验失败包含字段路径。
- Java 方法失败包含 methodKey 和错误摘要。
- AGENT_CALL 失败包含目标 Agent 和子运行编号，或无法创建子运行的原因。
- EXTERNAL_AGENT 失败包含适配器、退出码或 HTTP 状态、错误摘要。
- Eval 失败能定位到失败用例和失败断言。

## 6. 分页筛选

列表接口统一支持：

- page。
- pageSize。
- status。
- keyword。
- startedAtFrom / startedAtTo，适用于运行记录。

具体字段由接口设计文档展开。

## 7. OpenAPI 契约

- 后端 Controller 请求、响应、错误模型必须纳入 OpenAPI。
- 前端通过 OpenAPI 生成 API client 类型或请求响应类型。
- 不允许前端手工复制 Java DTO 字段作为长期契约。
- OpenAPI 文件是公共接口设计和前端类型生成的共同输入。
