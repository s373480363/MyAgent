# 13-部署安全与质量架构设计 v1

## 1. 部署边界

v1 面向本机或内网单用户部署，可以在信任环境内共享访问，但不提供用户隔离、租户隔离和公网安全边界。

## 2. 部署拓扑

```text
用户浏览器
  -> 前端静态资源服务
  -> Spring Boot API 服务
  -> PostgreSQL
  -> OpenAI 官方 API
  -> CLI / HTTP 外部 Agent，可选
```

## 3. 配置项

| 配置 | 来源 | 默认值 / 说明 |
|------|------|---------------|
| myagent.openai.api-key | 环境变量或本地配置 | OpenAI Key，不返回前端 |
| myagent.openai.default-model | 配置文件或系统设置 | 默认模型 |
| myagent.runtime.default-agent-timeout-seconds | 配置文件 | 600 |
| myagent.runtime.default-llm-timeout-seconds | 配置文件 | 120 |
| myagent.runtime.default-java-method-timeout-seconds | 配置文件 | 30 |
| myagent.runtime.default-external-agent-timeout-seconds | 配置文件 | 600 |
| myagent.runtime.default-max-steps | 配置文件 | 30 |
| myagent.runtime.default-max-agent-call-depth | 配置文件 | 3 |
| myagent.trace.persist-full-model-content | 配置文件 | true，控制模型提示词和模型输出是否持久保存全文 |

## 4. 运行数据保留

- v1 默认持久保存 AgentRun、NodeRun、TraceEvent。
- 模型提示词和模型输出默认保存全文，预留配置项控制开关。
- 不默认自动删除运行数据。
- 如后续需要清理策略，应先确认数据保留周期。

## 5. 安全边界

| 风险点 | 处理 |
|--------|------|
| 公网暴露 | v1 不按公网 SaaS 设计，部署说明标明仅用于本机或内网信任环境 |
| OpenAI Key 泄露 | Key 只在后端配置使用，不通过 API 返回 |
| 任意 Java 方法调用 | 只允许调用注解注册且启用的方法 |
| 条件表达式注入 | CONDITION 使用声明式规则，不执行脚本 |
| Agent 互调失控 | AGENT_CALL 限制最大调用深度，默认 3，禁止直接调用自己 |
| 外部 Agent 执行风险 | EXTERNAL_AGENT 必须通过适配器注册、默认超时、错误摘要接入 |
| 工具误调用 | TOOL 只能选择已注册且启用的工具，调用前后 Schema 校验 |
| 验收用例误计入 | AI_DRAFT_PENDING EvalCase 不计入正式通过率 |
| Trace 敏感内容 | 默认保存全文，依赖内网单用户部署边界；后续多人共享或公网部署需重做脱敏与访问控制 |

## 6. 测试策略

| 测试类型 | 覆盖对象 |
|----------|----------|
| 领域单元测试 | Agent 状态、WorkflowVersion 发布规则、Schema 锁定规则 |
| 运行内核测试 | 所有 v1 NodeExecutor 和失败路径 |
| Schema 测试 | 输入校验、输出校验、字段路径错误、版本引用 |
| 集成测试 | PostgreSQL、Flyway、MyBatis、运行记录、Trace 写入 |
| Agent 协作测试 | 父子 AgentRun、最大调用深度、子运行失败 |
| 外部 Agent 测试 | CLI/HTTP 适配器、超时、退出码、错误摘要 |
| 节点验收测试 | EvalSuite、EvalCase、EvalRun、断言和关键用例 |
| 前端测试 | 画布保存、节点配置、发布校验、运行详情 |
| 契约测试 | OpenAPI 契约生成、前端类型生成和关键接口兼容性 |

## 7. 编码规范影响

- 后端类、方法和字段必须按项目规范编写中文 Javadoc。
- 注释、日志、错误消息和 UI 文案使用中文。
- 不做双轨 Schema、双轨工作流定义、双轨运行记录。
- 抽象只在真实复用或明显降低复杂度时创建。
