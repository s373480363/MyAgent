# 04 运行时与注册目录接口 v1

## 1. 定位

本文件定义工作流运行、节点执行、Schema 校验、Trace 写入和注册目录访问的公共 Java 接口。

这些接口是平台执行面的骨架，重点在于稳定、可追踪和可替换。

## 2. WorkflowValidator

```java
public interface WorkflowValidator {
    ValidationResult validateDraft(WorkflowVersionSnapshot snapshot);
    ValidationResult validateForPublish(WorkflowVersionSnapshot snapshot);
}
```

职责：

- 校验节点、边、引用对象和运行限制。
- 发布前阻止非法工作流进入正式版本。
- 返回字段级问题，而不是只给一个总失败。

## 3. WorkflowCompiler

```java
public interface WorkflowCompiler {
    CompiledWorkflow compile(WorkflowVersionSnapshot snapshot);
}
```

职责：

- 将工作流版本编译为可执行图。
- 绑定节点执行器、条件边和结束边。
- 不修改原始草稿或发布快照。

## 4. NodeExecutorRegistry

```java
public interface NodeExecutorRegistry {
    NodeExecutor getExecutor(NodeType nodeType);
    boolean supports(NodeType nodeType);
}
```

职责：

- 根据节点类型找到对应执行器。
- 统一管理节点执行能力。
- 不允许页面或工作流直接指定任意类名绕过注册。

## 5. NodeExecutor

```java
public interface NodeExecutor {
    NodeExecutionResult execute(NodeExecutionContext context);
}
```

职责：

- 完成节点专属执行逻辑。
- 执行前后进行必要的 Schema 校验。
- 产出节点输出、Trace 事件和错误信息。

## 6. NodeExecutionContext

建议字段：

- `agentRunDbId`
- `agentRunNo`
- `nodeRunDbId`
- `agentId`
- `agentKey`
- `workflowVersionId`
- `parentRunId`
- `nodeDefinition`
- `agentDefinition`
- `workflowContext`
- `runtimeOptions`
- `traceWriter`
- `schemaValidationService`
- `mappingService`
- `runtimeLimitGuard`

说明：

- `agentRunDbId` 对应数据库 `agent_run.id`，用于运行时和持久化层内部关联。
- `agentRunNo` 对应数据库 `agent_run.run_no`，用于对外响应和日志展示。
- `nodeRunDbId` 对应数据库 `node_run.id`，用于运行时和持久化层内部关联；对外接口中的 `nodeRunId` 也统一使用该数据库主键。
- `nodeRunDbId` 由 Runtime 在调用 NodeExecutor 之前通过 `TraceWriter.createNodeRun` 创建。
- NodeExecutor 只使用已有 `nodeRunDbId` 写事件和完成状态，不负责创建 NodeRun。
- `runtimeOptions` 对应已持久化 `WorkflowVersion.runtime_options_json`，正式字段集固定为 `timeoutSeconds`、`maxSteps`、`maxAgentCallDepth`。
- 运行时执行只能读取该快照对象，不得在执行阶段回头读取 Agent 当前主数据重算总超时或最大步数。

## 7. NodeExecutionResult

建议字段：

- `status`
- `outputJson`
- `schemaValidationResult`
- `selectedEdgeId`
- `errorCode`
- `errorMessage`
- `traceEvents`
- `durationMs`

规则：

- 成功和失败都应尽量返回可定位信息。
- 条件节点应显式返回选中的分支。
- 失败时必须给出中文摘要。

## 8. MappingService

```java
public interface MappingService {
    JsonNode extractInput(JsonNode workflowContext, MappingDefinition inputMapping);
    JsonNode applyOutput(JsonNode workflowContext, MappingDefinition outputMapping, JsonNode nodeOutput);
}
```

职责：

- 处理输入映射。
- 处理输出写回。
- 仅支持受控 JSONPath 子集。

规则：

- 不支持任意脚本。
- 不支持过滤表达式和函数调用。
- 映射失败应视为节点失败。

## 9. SchemaValidationService

```java
public interface SchemaValidationService {
    SchemaValidationResult validate(JsonNode payload, SchemaReference schemaRef, ValidationStage stage);
}
```

职责：

- 校验输入、输出和中间结构。
- 返回字段路径和中文错误。
- 支持不同阶段的上下文提示。

## 10. TraceWriter

```java
public interface TraceWriter {
    NodeRunStartResult createNodeRun(NodeRunStartRecord record);
    void finishNodeRun(NodeRunFinishRecord record);
    void writeEvent(TraceEventRecord record);
}
```

```java
public interface NodeRunStartResult {
    Long getNodeRunDbId();
    Long getAgentRunDbId();
    String getAgentRunNo();
    String getNodeId();
    Instant getStartedAt();
}
```

职责：

- 写入 NodeRun。
- 写入 TraceEvent。
- 保证失败时尽力落库。

规则：

- Runtime 进入节点前调用 `createNodeRun`，并将返回的 `nodeRunDbId` 放入 NodeExecutionContext。
- NodeExecutor 结束后或异常捕获后，由 Runtime 调用 `finishNodeRun`。
- `NodeRunStartResult` 至少包含 `nodeRunDbId`、`agentRunDbId`、`agentRunNo`、`nodeId` 和 `startedAt`。
- 不应把整个 Trace 只存在内存。
- `summary` 和 `detailJson` 分离存储。

## 11. RuntimeLimitGuard

```java
public interface RuntimeLimitGuard {
    void checkRunTimeout(RunLimitContext context);
    void checkNodeTimeout(RunLimitContext context);
    void checkStepLimit(RunLimitContext context);
    void checkCallDepth(RunLimitContext context);
}
```

职责：

- 控制运行超时。
- 控制步数上限。
- 控制 AGENT_CALL 深度。

## 12. ModelOfferingRegistry

```java
public interface ModelOfferingRegistry {
    ModelOfferingDescriptor resolve(String modelOfferingKey);
    List<ModelOfferingDescriptor> listEnabled(ModelOfferingQuery query);
    List<ModelOfferingDescriptor> findByOfferingKeys(List<String> offeringKeys);
}
```

职责：

- 管理模型供应商和模型供应项。
- 为 LLM 类节点提供 `modelOfferingKey` 解析。
- 确保运行时只调用已启用供应商下的已启用供应项。
- `ModelOfferingDescriptor` 是可返回前端和可写 Trace 的安全描述，不包含 Base URL、API Key 或密文。

## 13. ModelGateway

```java
public interface ModelGateway {
    ModelInvocationResult invoke(ModelInvocationRequest request);
}
```

职责：

- 封装 OpenAI-compatible 模型供应商调用。
- `ModelInvocationRequest` 只携带 `modelOfferingKey`、提示词、输入、温度和结构化输出标记。
- 网关内部通过 `modelOfferingKey` 解析 `ResolvedModelRoute`，该内部对象可以包含 Base URL 和解密 API Key，但不得写入 Trace、日志、REST DTO 或前端状态。
- MODEL_REQUEST Trace 必须使用 `ModelRequestTracePayload` 等白名单 DTO 构造，禁止直接序列化 `ModelInvocationRequest` 或 `ResolvedModelRoute`。
- 支持普通文本输出和结构化输出。
- 返回 token、耗时和失败摘要。

## 14. JavaMethodRegistry

```java
public interface JavaMethodRegistry {
    Optional<JavaMethodDescriptor> findByMethodKey(String methodKey);
    List<JavaMethodDescriptor> listEnabled();
    JavaMethodInvoker getInvoker(String methodKey);
}
```

## 15. JavaMethodInvoker

```java
public interface JavaMethodInvoker {
    JsonNode invoke(JsonNode input);
}
```

职责：

- 将 JSON 入参转换为方法可执行边界。
- 调用 Spring Bean 上的显式注册方法。
- 返回可序列化 JSON。

规则：

- 不允许直接反射任意方法。
- 入参出参必须经过 Schema 校验。

public interface ToolRegistry {

    List<ToolDescriptor> listEnabled();
    ToolExecutor getExecutor(String toolKey);

职责：

- 提供平台工具目录。
- 执行已注册工具。

## 21. 运行时接口设计结论

运行时接口只做三件事：

1. 把输入变成可执行上下文。
2. 把执行过程变成可追踪事件。
3. 把结果变成稳定的运行记录。

任何会破坏这三件事的做法，都不应进入运行时接口层。
