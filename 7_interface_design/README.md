# 7 公共接口设计总览 v1

## 1. 目录目标

本目录定义平台 v1 的公共接口设计，覆盖两类边界：

- 对外 REST API。
- 对内公共 Java 接口，包括应用服务、运行时、注册目录和适配器接口。

本目录的目标是让后端 Controller、前端类型、工作流运行、节点执行、外部适配器和验收流程共享一致契约，避免双轨真相。

## 4. 设计边界

- 对外 REST 只暴露稳定业务契约。
- 内部服务接口是 Controller、定时任务、导入导出和运行编排的共同入口。
- Runtime 接口只负责执行，不直接处理页面交互。
- Registry 接口只负责目录与查找，不承载业务流程。
- 所有用户可见错误必须使用中文。
