```powershell

```

结果：

### 3.2 旧变量 fail-fast 验证

执行命令：

```powershell
cd D:\myproject\MyAgent
$env:AGENT_STUDIO_POSTGRES_PASSWORD='agent_studio_dev_password'
$env:AGENT_STUDIO_OPENAI_API_KEY='present'
docker compose run --rm -e OPENAI_API_KEY=invalid-old-key api
```

结果：

- 按预期失败。
- 启动日志明确提示：检测到已废弃的 `OPENAI_API_KEY`，要求改用 `AGENT_STUDIO_OPENAI_API_KEY`。

### 3.3 正式部署启动与接口验收

执行命令：

```powershell
cd D:\myproject\MyAgent
$env:AGENT_STUDIO_POSTGRES_PASSWORD='agent_studio_dev_password'
$env:AGENT_STUDIO_OPENAI_API_KEY='<real-key>'
docker compose up -d --build
curl.exe -sS http://127.0.0.1:18080/actuator/health
curl.exe -sS http://127.0.0.1:18080/api/settings
Invoke-RestMethod -Uri 'http://127.0.0.1:18080/v3/api-docs'
docker compose exec postgres psql -U agent_studio -d agent_studio -c "select version from flyway_schema_history order by installed_rank;"
docker compose exec postgres psql -U agent_studio -d agent_studio -c "select setting_key from system_setting order by setting_key;"
```

结果：

- `postgres`、`api`、`web` 三个服务均成功启动。
- `/actuator/health` 返回 `{"status":"UP"}`。
- `/api/settings` 返回 7 个白名单设置项，来源均为 `APPLICATION_CONFIG`。
- `/v3/api-docs` 返回 `servers[0].url = http://127.0.0.1:18080`。
- `flyway_schema_history` 版本为 `1, 2, 3`。
- `system_setting` 为空表，符合 fresh install 验收口径。
