## 5. 待整改问题

### P1-01 Docker 初始 Poe base URL 会导致真实模型调用失败

现象：按测试环境启动后，默认供应商 `baseUrl=https://api.poe.com/v1`，LLM 节点真实调用失败。API 容器日志显示 Poe 返回 `404 Not Found`。  
验证：同一密钥直连 `https://api.poe.com/v1/chat/completions` 可成功；通过正式接口把供应商 base URL 改为 `https://api.poe.com` 后，模型供应商连接测试和真实 LLM/EvalRun 均成功。  
影响：正式 Docker 初始环境无法直接完成真实 LLM 验收，必须人工修正供应商 base URL。  
建议：修正 Docker env / seed 逻辑，明确 Spring AI 传入的 base URL 应为 `https://api.poe.com`，避免重复拼接 `/v1`。

### P1-02 前端 OpenAPI generated schema 未同步

现象：`npm run openapi:check` 发现 `11_code/frontend/src/api/generated/schema.ts` 与正式 OpenAPI 不一致，并在检查过程中重生成该文件。  
证据：git diff 显示 `referenceAnswer/assertions/scoreRule` 被替换为 `referenceSample/judgeRule/hardChecks`，并新增 `EvalHardCheckResultItem/judgeResult/judgeRawText` 等字段。  
影响：契约生成物未提交会导致 CI 或其他开发环境拿到旧类型定义。  
建议：提交最新 generated schema，重新执行 `npm run openapi:check` 并确保无 diff。
