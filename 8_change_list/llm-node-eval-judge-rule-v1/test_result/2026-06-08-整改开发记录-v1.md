# 2026-06-08 整改开发记录 v1

## 1. 本轮整改范围

本轮针对两轮真实验收报告中的正式待整改项进行开发修复，范围包括：

- 修正默认供应商初始化时的 Poe Base URL 风险。
- 收口前端 OpenAPI generated schema。
- 清理正式文档中的旧“参考答案 / 断言 / 评分”语义。
- 补齐节点验收用户手册。

本轮不重写 Eval judge 主链，不调整正式 Eval 字段契约。

## 2. 已完成修复

### 2.1 默认供应商初始化 Base URL 收口

已修改：

- `11_code/backend/src/main/java/com/myagent/modelcatalog/application/ModelCatalogBootstrapDefaults.java`
- `11_code/backend/src/test/java/com/myagent/modelcatalog/application/ModelCatalogBootstrapDefaultsTests.java`

收口结果：

- 默认供应商初始化时，如果环境变量传入 `https://api.poe.com/v1` 或 `https://api.poe.com/v1/`，会归一化为 `https://api.poe.com`。
- 普通 OpenAI-compatible Base URL 保持不变。
- 空值仍回退到正式默认值 `https://api.openai.com`。

### 2.2 OpenAPI generated schema

已收口：

- `11_code/frontend/src/api/generated/schema.ts`

- generated schema 与当前正式 OpenAPI 一致。
- `referenceSample`、`judgeRule`、`hardChecks`、`hardCheckResults`、`judgeRawText` 等新契约已固定到生成物。

### 2.3 正式文档术语清理

## 3. 预期复验项

建议按以下顺序复验：

1. 执行后端 `mvn clean test`。
2. 执行前端 `npm test -- --run`、`npm run build`、`npm run openapi:check`。
3. 执行正式文档静态检索，确认旧术语已清除。
4. 如需彻底关闭初始化风险，再清空 Docker volume 后重建正式入口，验证默认供应商 Base URL 初始化结果。
