## 3. 2026-06-04 首次正式入口尝试（历史记录）

### 3.1 Docker 正式入口启动

执行：

```powershell
cd D:\myproject\MyAgent
$env:AGENT_STUDIO_POSTGRES_PASSWORD='agent_studio_dev_password'
$env:AGENT_STUDIO_OPENAI_API_KEY='dummy-key'
docker compose up -d --build
```

结果：

- `postgres`、`web` 容器启动。
- `api` 容器未保持运行。

### 3.2 正式入口健康检查

执行：

```powershell
curl.exe -sS http://127.0.0.1:18080/actuator/health
curl.exe -sS http://127.0.0.1:18080/api/settings
curl.exe -sS http://127.0.0.1:18080/v3/api-docs
```

结果：

- 三个地址都返回 `502 Bad Gateway`。
- 这说明正式 Web 入口已起来，但上游 API 未成功启动。

### 3.3 阻塞定位

执行：

```powershell
$env:AGENT_STUDIO_POSTGRES_PASSWORD='agent_studio_dev_password'
$env:AGENT_STUDIO_OPENAI_API_KEY='dummy-key'
docker compose ps
docker compose logs --tail=200 api
docker compose logs --tail=100 web
```

结果：

- `docker compose ps` 中只剩 `postgres` 和 `web` 保持运行。
- `api` 日志显示 Flyway/数据源初始化失败，核心错误为：
  - `FATAL: password authentication failed for user "agent_studio"`
- `web` 日志显示对 `http://api:8080` 的反向代理连接被拒绝，因而返回 `502`。

结论：

- 该次尝试受既有 `postgres-data` 数据卷中的历史数据库密码阻塞。
- 这是一次带历史卷状态的环境问题记录，不代表当前 fresh install 的正式入口状态。
