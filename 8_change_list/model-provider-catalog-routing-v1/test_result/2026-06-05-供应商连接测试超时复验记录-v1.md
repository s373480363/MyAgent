# 模型供应商连接测试超时复验记录

## 1. 复验背景

在阅读最新实现与验收文档后，针对 `POST /api/model-providers/{providerId}/test` 做了独立复验。复验目标不是验证“失败消息是否脱敏”这一点本身，而是确认该接口是否真的满足正式契约中的“连接测试”语义：轻量、有限时、可预测返回。

## 2. 复验环境

- 正式入口：`http://127.0.0.1:18080`
- 健康检查：`/actuator/health` 返回 `{"status":"UP"}`
- 复验时间：`2026-06-05`

## 3. 复验命令

请求体：

```json
{
  "offeringKey": "openai.gpt_4_1_mini",
  "prompt": "ping"
}
```

第一次复验：

```powershell
Invoke-RestMethod -Method Post `
  -Uri 'http://127.0.0.1:18080/api/model-providers/1/test' `
  -ContentType 'application/json' `
  -Body '{"offeringKey":"openai.gpt_4_1_mini","prompt":"ping"}' `
  -TimeoutSec 15
```

结果：

- 调用方 15 秒超时。
- 返回 `System.Net.WebException`
- 消息：`The request was aborted: The operation has timed out.`

第二次复验：

```powershell
Invoke-RestMethod -Method Post `
  -Uri 'http://127.0.0.1:18080/api/model-providers/1/test' `
  -ContentType 'application/json' `
  -Body '{"offeringKey":"openai.gpt_4_1_mini","prompt":"ping"}' `
  -TimeoutSec 60
```

结果：

- 约 60 秒后返回 `504 Gateway Timeout`
- 调用方异常类型仍为 `System.Net.WebException`

## 4. 代码对照

复验结果与当前实现一致：

- `DefaultModelCatalogApplicationService.testModelProvider(...)` 直接调用 `openAiCompatibleModelInvoker.invoke(...)`
  - `11_code/backend/src/main/java/com/myagent/modelcatalog/application/DefaultModelCatalogApplicationService.java`
- `OpenAiCompatibleModelInvoker.createChatModel(...)` 只按 `baseUrl` 和 `apiKey` 创建 `OpenAiApi/OpenAiChatModel`，没有显式超时边界
  - `11_code/backend/src/main/java/com/myagent/model/OpenAiCompatibleModelInvoker.java`
- 正式工作流节点执行存在外层 `Future#get(timeout)` 包装
  - `11_code/backend/src/main/java/com/myagent/runtime/NodeExecutionRunner.java`
- 供应商连接测试当前没有走同类超时包装，因此不能证明它复用了正式 `agent.studio.runtime.default-llm-timeout-seconds` 契约

## 5. 复验结论

当前实现下：

- 已验证服务存活。
- 已验证连接测试失败时不会直接回显 API Key。
- 但没有验证通过“连接测试是有限时探活接口”这一正式契约。

因此本接口当前不能认定为完整验收通过，原因如下：

1. 页面“连接测试”请求会长时间等待。
2. 超时边界没有与正式 LLM 调用契约收口。
3. 失败是否在统一超时窗口内返回，当前取决于外层网关或调用方，而不是系统自己的正式行为。

## 6. 建议处理

- 将供应商连接测试纳入统一超时包装。
- V1 直接复用 `agent.studio.runtime.default-llm-timeout-seconds`，不要再新增第二套测试超时配置。
- 补充后端专用测试，至少覆盖：
  - 成功路径
  - 超时中断路径
  - 中文错误摘要
  - 不泄露 API Key / Base URL / 上游完整鉴权报文

在上述修复完成前，本变更只能认定为“主体能力已完成，但连接测试契约仍有阻塞缺口”。
