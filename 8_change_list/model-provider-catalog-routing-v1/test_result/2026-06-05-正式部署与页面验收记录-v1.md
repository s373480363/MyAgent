## 3. API 验收

### 3.1 模型目录接口

执行：

```powershell
Invoke-RestMethod -Uri 'http://127.0.0.1:18080/api/model-providers?page=1&pageSize=20'
Invoke-RestMethod -Uri 'http://127.0.0.1:18080/api/model-offerings?page=1&pageSize=20'
Invoke-RestMethod -Uri 'http://127.0.0.1:18080/api/model-offerings/by-keys?offeringKeys=openai.gpt_4_1_mini'
```

结果：

- 供应商列表返回分页结构。
- 响应仅包含 `apiKeyConfigured` / `apiKeyMask`，不返回明文 API Key。
- 模型供应项列表与 `by-keys` 返回正式 `modelOfferingKey` 事实。

### 3.2 `by-keys` 回填禁用供应项

执行：

```powershell
Invoke-RestMethod -Method Put -Uri 'http://127.0.0.1:18080/api/model-offerings/2/status' -ContentType 'application/json' -Body '{"status":"DISABLED"}'
Invoke-RestMethod -Uri 'http://127.0.0.1:18080/api/model-offerings/by-keys?offeringKeys=browser.gpt_4_1_mini'
Invoke-RestMethod -Method Put -Uri 'http://127.0.0.1:18080/api/model-offerings/2/status' -ContentType 'application/json' -Body '{"status":"ENABLED"}'
```

结果：

- `by-keys` 在供应项停用后仍返回该记录。
- 返回值中 `selectable=false`，`unavailableReason=模型供应项已停用`。
- 验证了“当前绑定可回填、可选列表可禁用”的正式契约。

## 4. 页面验收

正式入口：

```text
http://127.0.0.1:18080
```

### 4.1 模型供应商页面

- 打开 `/model-providers`。
- 通过页面创建 `browser-provider`。
- 在供应商详情页通过页面创建 `browser.gpt_4_1_mini`。
- 页面仅显示 `bro-...-key` 掩码，不回显密钥明文。

### 4.2 Agent 页面

- 打开 `/agents`。
- 通过页面创建 `browser-agent`。
- 在创建弹窗中选择默认模型供应项 `browser.gpt_4_1_mini`。
- 创建后的 Agent 详情显示 `defaultModelOfferingKey=browser.gpt_4_1_mini`。

### 4.3 工作流页面

- 打开 `/agents/1/workflow`。
- 通过页面添加 `START`、`LLM`、`END` 三个节点。
- 选中 `LLM` 节点，通过页面模型供应项选择器选择 `browser.gpt_4_1_mini`，填写 `userPromptTemplate`，并应用节点属性。
- 通过页面保存草稿后，刷新页面重新打开，`LLM` 节点仍正确回填 `browser.gpt_4_1_mini`。
- 为构造可发布的最小工作流，复用现有 `/api/schemas` 与 `/api/agents/{agentId}/workflow-draft` 接口补全：
  - `START.inputSchemaRef`
  - `END.outputSchemaRef`
  - `START -> LLM -> END` 边关系
- 在页面再次点击“发布工作流”并确认，页面提示 `工作流已发布。`

### 4.4 页面与运行事实核对

执行：

```powershell

Invoke-RestMethod -Uri 'http://127.0.0.1:18080/api/agents/1'
```

结果：

- 当前草稿 `nodes[*].config` 中保存的是 `modelOfferingKey=browser.gpt_4_1_mini`，不含旧 `model` 字段。
- `currentPublishedWorkflow.workflowVersionId` 已更新为页面发布产生的新版本。
