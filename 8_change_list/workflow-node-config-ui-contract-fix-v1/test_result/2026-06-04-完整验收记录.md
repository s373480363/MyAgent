## 源码复核记录

- `WorkflowPage.tsx` 仍通过 `configText`、`inputSchemaRefText`、`outputSchemaRefText` 承载节点配置。
- 后端 LLM/REVIEW/SUMMARY 执行器实际读取 `model`、`temperature`，公共支持类读取 `userPromptTemplate`、`systemPromptTemplate`。
- 后端发布校验要求 LLM 类节点必须有 `userPromptTemplate`。
- 后端发布校验要求目录型节点配置 `methodKey`、`toolKey`、`adapterKey`、`targetAgentKey`。
- 后端发布校验要求 CONDITION 分支边配置条件对象。
- 前端工作流页面没有使用已有的目录列表接口来渲染节点选择器。

详细问题和证据见：`../audit_findings.md`。

## 验收结论

当前实现不符合 `workflow-node-config-ui-contract-fix-v1` 的预期。

阻断原因：

- 工作流 LLM 类节点仍没有显式模型、温度、提示词表单。
- CONDITION 分支条件没有边级配置 UI。
- 目录型节点没有目录选择器。
- Schema 引用没有 Schema 选择器。

非阻断但需要修复的问题：

- Agent 创建页模型字段语义不清。
- 评测 LLM 评分规则仍是 JSON-only。
- 评测套件要求手填 Agent ID、WorkflowVersion ID 和节点 ID。
- 外部 Agent 命令或 HTTP 配置仍是 JSON-only。
- 外部 Agent 输出 Schema 要求手填数字 ID。
- 工作流页面测试覆盖不足。
- `openapi:check` 脚本失败时会留下生成文件变更。
