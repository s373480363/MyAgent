## 发现的问题

### P2-01 目录选择器会把任何未加载详情的已有绑定值显示成“已绑定失效项”

证据：

- `WorkflowPage.tsx` 中 `RemotePagedSelect` 只要 `value` 存在就创建 `boundFallbackOption`。
- Java 方法、工具、Agent、外部 Agent、Schema 的 `boundValueFormatter` 文案均使用“已绑定失效项”。

影响：

如果一个已有草稿引用的 key 实际仍然有效，但当前选择器尚未打开、尚未加载第一页，或者当前搜索结果尚未命中该 key，UI 也会显示“已绑定失效项”。这会把“未加载详情 / 当前搜索未命中”和“真实失效”混为一谈，容易误导用户。

预期处理：

- 保留原值的策略是正确的，不能回退。
- 但文案应改成更准确的状态，例如 `当前绑定：xxx（未加载详情）` 或 `当前绑定：xxx（当前结果未命中）`。
- 只有在前端已经有明确证据确认对象未启用、未发布或不存在时，才显示“失效”。
